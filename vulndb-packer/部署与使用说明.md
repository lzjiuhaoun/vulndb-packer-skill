# VulnDB Packer 部署与使用说明

## 一、概述

VulnDB Packer 是一个漏洞库升级包制作工具，用于根据用户提供的材料自动生成漏洞库升级包。

### 功能特性

- 解析 MySQL SQL 脚本，提取 INSERT 语句
- 将 MySQL SQL 转换为达梦8 SQL
- 读取 Excel 文件，提取漏洞变化统计信息
- 生成升级包 zip 文件
- 生成安全漏洞规则库变化说明（包含漏洞详情）
- 版本号校验（格式和大小）
- 升级包验证

## 二、环境要求

### 2.1 操作系统

- Windows 10/11
- Windows Server 2016 及以上

### 2.2 软件要求

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| Python | 3.8+ | 推荐 3.10 |
| Microsoft Excel | 2016+ | 用于读取加密的 Excel 文件 |
| pywin32 | 306+ | Python COM 接口库 |

### 2.3 硬件要求

- 内存：4GB+（处理大文件时建议 8GB+）
- 磁盘空间：1GB+（用于存储生成的升级包）

## 三、部署步骤

### 3.1 安装 Python

1. 下载 Python 3.10+：https://www.python.org/downloads/
2. 安装时勾选 "Add Python to PATH"
3. 验证安装：
```bash
python --version
```

### 3.2 安装依赖

```bash
pip install pywin32
```

### 3.3 部署 VulnDB Packer

#### 方式一：作为 Agent Skill 部署

1. 将 `vulndb-packer` 目录复制到 Agent 的 skill 目录下
2. 目录结构：
```
~/.agents/skills/
└── vulndb-packer/
    ├── SKILL.md
    ├── README.md
    ├── scripts/
    │   ├── main.py
    │   ├── sql_parser.py
    │   ├── sql_converter.py
    │   ├── excel_reader.py
    │   ├── excel_extractor.py
    │   ├── package_builder.py
    │   ├── report_generator.py
    │   └── logger.py
    ├── templates/
    └── evals/
```

3. Agent 会自动识别 SKILL.md 并根据用户需求调用脚本

#### 方式二：独立部署

1. 将 `vulndb-packer` 目录复制到任意位置
2. 通过命令行直接调用：
```bash
python vulndb-packer/scripts/main.py [参数]
```

## 四、使用说明

### 4.1 输入文件准备

#### MySQL SQL 文件 (va_library.sql)

- 文件格式：.sql
- 编码：UTF-8
- 内容：INSERT 语句
- 示例：
```sql
INSERT INTO `va_library` VALUES (1, 'CVE-2024-1234', '2024-01-15 10:30:00', 1, 1, 'check_rule_1', 1, 'MySQL权限提升漏洞', 'ALTER USER root IDENTIFIED BY new_password;', 3, 1, '建议升级MySQL到最新版本', 'CVE-2024-1234', 'CNNVD-202401-1234', 'MySQL 5.7', 'V1.0.2.19', 1);
```

#### Excel 文件

- 文件格式：.xls 或 .xlsx
- 文件名建议格式：`YYYY-MM漏扫数据库变化.xlsx`
- 示例：`2026-05漏扫数据库变化.xlsx`
- 内容格式：
```
共调整40个
1 新增漏洞40个：
    1.1 DB2新增：9个
    1.2 MySQL新增：11个
    ...
2 修改漏洞0个：
3 删除漏洞0个：
```

#### 版本号

- 格式：`Vx.y.z.w`（如 V1.0.2.19）
- 规则：新版本号必须大于当前版本号

### 4.2 命令行使用

#### 基本用法

```bash
python vulndb-packer/scripts/main.py \
  --mysql-sql "path/to/va_library.sql" \
  --excel-files "path/to/2026-05漏扫数据库变化.xlsx" \
  --current-version "V1.0.2.19" \
  --new-version "V1.0.2.20" \
  --output-dir "path/to/output"
```

#### 多个 Excel 文件

```bash
python vulndb-packer/scripts/main.py \
  --mysql-sql "path/to/va_library.sql" \
  --excel-files "path/to/2026-04漏扫数据库变化.xlsx" "path/to/2026-05漏扫数据库变化.xlsx" \
  --current-version "V1.0.2.19" \
  --new-version "V1.0.2.20" \
  --output-dir "path/to/output"
```

#### 参数说明

| 参数 | 说明 | 必需 |
|------|------|------|
| --mysql-sql | MySQL SQL 脚本文件路径 | 是 |
| --excel-files | Excel 文件路径列表 | 是 |
| --current-version | 当前漏洞库版本号 | 是 |
| --new-version | 新漏洞库版本号 | 是 |
| --output-dir | 输出目录 | 否（默认当前目录） |

### 4.3 Agent 使用

将需求告诉 Agent 即可：

```
请使用vulndb-packer skill制作漏洞库升级包：
- MySQL SQL文件：va_library.sql
- Excel文件：2026-04漏扫数据库变化.xlsx、2026-05漏扫数据库变化.xlsx
- 当前版本：V1.0.2.19
- 新版本：V1.0.2.20
```

### 5.2 升级包内容 (VSLib{YYYYMMDD}_{序号}.zip)

| 文件 | 说明 |
|------|------|
| source.txt | 版本信息文件 |
| va_library.sql | MySQL SQL 脚本（包含表结构定义） |
| va_library_dm.sql | 达梦8 SQL 脚本 |

#### source.txt 示例

```txt

# 漏洞库版本发布日期

```

#### va_library.sql 示例

```sql
DROP TABLE IF EXISTS `va_library`;
CREATE TABLE `va_library`  (
  `va_id` int(11) UNSIGNED NOT NULL AUTO_INCREMENT,
  `va_name` varchar(255) CHARACTER SET utf8 COLLATE utf8_general_ci NULL DEFAULT NULL COMMENT '漏洞名称',
  ...
  PRIMARY KEY (`va_id`) USING BTREE
) ENGINE = InnoDB CHARACTER SET = utf8 COLLATE = utf8_general_ci COMMENT = '漏洞库信息表' ROW_FORMAT = Dynamic;
INSERT INTO `va_library` VALUES (...);
```

#### va_library_dm.sql 示例

```sql
set define off;
TRUNCATE TABLE "AAS_VS"."VA_LIBRARY";
SET IDENTITY_INSERT "AAS_VS"."VA_LIBRARY" ON;
INSERT INTO "AAS_VS"."VA_LIBRARY" ("VA_ID", "VA_NAME", ...) VALUES (...);
SET IDENTITY_INSERT "AAS_VS"."VA_LIBRARY" OFF;
commit;
exit;
```

## 七、错误处理

### 7.1 常见错误及解决方法

| 错误信息 | 原因 | 解决方法 |
|----------|------|----------|
| MySQL SQL文件不存在 | 文件路径错误 | 检查文件路径是否正确 |
| Excel文件不存在 | 文件路径错误 | 检查文件路径是否正确 |
| 不支持的Excel文件格式 | 文件格式错误 | 使用 .xls 或 .xlsx 格式 |
| 当前版本格式不正确 | 版本号格式错误 | 使用 Vx.y.z.w 格式 |
| 新版本号必须大于当前版本号 | 版本号大小错误 | 确保新版本号大于当前版本号 |
| 所有Excel文件读取失败 | COM 接口问题 | 确保安装了 Microsoft Excel |
| 升级包文件不存在 | 生成失败 | 检查日志了解详细错误 |

### 7.2 查看日志

执行过程中会输出详细日志，包括：
- 执行步骤
- 错误信息
- 执行结果

## 八、注意事项

### 8.1 平台要求

- 仅支持 Windows 平台
- 需要安装 Microsoft Excel

### 8.2 文件格式

- MySQL SQL 文件：UTF-8 编码
- Excel 文件：.xls 或 .xlsx 格式
- 文件名建议：`YYYY-MM漏扫数据库变化.xlsx`

### 8.3 版本号规则

- 格式：`Vx.y.z.w`（如 V1.0.2.19）
- 必须以 `V` 开头
- 每段都是数字
- 新版本号必须大于当前版本号

### 8.4 换行符

生成的 SQL 文件使用 Unix (LF) 换行符

### 8.5 大文件处理

- 处理大 SQL 文件时可能需要较长时间
- 建议内存 8GB+

## 九、故障排除

### 9.1 pywin32 安装失败

```bash
# 尝试使用镜像源
pip install pywin32 -i https://pypi.tuna.tsinghua.edu.cn/simple

# 或者手动下载安装
# 下载地址：https://github.com/mhammond/pywin32/releases
```

### 9.2 COM 接口初始化失败

1. 确保安装了 Microsoft Excel
2. 确保 Excel 可以正常打开
3. 尝试以管理员权限运行

### 9.3 Excel 文件读取失败

1. 确保文件未被其他程序占用
2. 确保文件未损坏
3. 尝试用 Excel 打开并重新保存

### 9.4 版本号校验失败

1. 检查版本号格式是否正确（Vx.y.z.w）
2. 检查新版本号是否大于当前版本号
3. 检查版本号是否包含特殊字符
